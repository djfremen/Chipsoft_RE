CanScan - сканер CAN шины на базе CHIPSOFT J2534 устройства.

Для корректной работы программы CanScan v0.5 необходимо наличие прошивки версии не ниже 0.95 CHIPSOFT J2534 устройства

Программа работает через командную строку.

Если возникла необходимость прервать работу программы, делайте это с помощью сочетания клавиш Ctrl-C. В противном случае CHIPSOFT J2534 устройство корректно не закрется и потребуется его переподключение для последующей нормальной работы.

Ключи программы:
-all Получение всех без исключения CAN пакетов
-new Получение всех новых CAN пакетов (значение по умолчанию)
-b[Value] Задание скорости обмена CAN шины, например, -b500000
-o[Filename] Запись полученных CAN пакетов во внешний файл, например, -ocanlog.txt
-rw Перезаписывать каждый раз внешний файл с записанными CAN пакетами
-mf[MaskFilter] Задать mask filter (hex), например, -mf000012AB
-pf[PatternFilter] Задать pattern filter (hex), например -pf000012AB
-rdt[Timeout] Период опроса CAN шины программой, например, -rdt1000
-wrt[Timeout] Максимальное время (мс) отправки CAN сообщения, например, -wrt1000
-per[Time] Период (мс) отправки CAN сообщений, например, -per100
-wcnt[Number] Количество CAN сообщений на для отправки, например, -wcnt10
-txf[Value] Задание флага передачи CAN сообщения (hex), например, -txf40
-msg[Message] Задание CAN сообщения (hex) на отправку, например, -msg000012341122334455667788
-rcnt[Number] Получение ответа (количество CAN сообщений) на отправленное CAN сообщение, например, -rcnt2
-sil Не выводить вспомагательную информацию на экран
-noscreen Не выводить CAN пакеты на экран (рекомендуется использовать при работе с высоконагруженными CAN шинами)

Значение по-умолчанию, если ключи не заданы:
Скорость обмена CAN шины = 500000
MaskFilter = 0x00000000
PatternFilter = 0x00000000
Период опроса CAN шины программой в цикле (мс) = 1000

Примеры работы с программой CanScan

Режим чтения CAN шины
---------------------

1. Получение всех новых CAN пакетов (повторяющиеся не выводятся), ключ -new
>canscan.exe -new

2. Получение всех без исключения CAN пакетов, ключ -all
>canscan.exe -all

3. Запись полученных CAN пакетов во внешний файл, ключ -o[имя_файла]
>canscan.exe -all -olog.txt
В данном примере осуществляется запись в файл log.txt

4. Задание скорости обмена CAN шины, ключ -b[скорость]
>canscan.exe -all -olog.txt -b500000
В данном примере CAN шина настраивается на скорость 500000 bits/s

5. Не выводить вспомагательную информацию на экран, ключ -sil
>canscan.exe -all -olog.txt -b500000 -sil

6. Перезаписывать каждый раз внешний файл с записанными CAN пакетами, ключ -rw
>canscan.exe -all -olog.txt -b500000 -sil -rw

7. Период опроса CAN шины программой в цикле (мс), ключ -rdt[время]
>canscan.exe -rdt1000
В данном примере опрос проводится каждую секунду

9. Задание фильтров для сканирования CAN шины (hex), mask filter, pattern filter, ключи -mf[значение], -pf[значение]
>canscan.exe -mfFFFFFFFF -pf00000510
В данном примере ловятся только CAN пакеты с идентификатором 00000510

Режим записи в CAN шину
-----------------------

1. Задание CAN сообщения (hex) на отправку, ключ -msg[сообщение]
>canscan.exe -msg000012341122334455667788
В данном примере отправляется сообщение [00001234] 11 22 33 44 55 66 77 88 на скорости, по-умолчанию

2. Максимальное время (мс) отправки CAN сообщения, ключ -wrt[время]
>canscan.exe -msg000012341122334455667788 -wrt200

3. Количество CAN сообщений на для отправки, ключ -wcnt[количество]
>canscan.exe -msg000012341122334455667788 -wrt200 -wcnt50

4. Период (мс) отправки CAN сообщений, ключ -per[значение]
>canscan.exe -msg000012341122334455667788 -wrt200 -wcnt50 -per200

5. Задание флага передачи CAN сообщения (hex), ключ -txfC0
>canscan.exe -msg000012341122334455667788 -wrt200 -wcnt50 -per200 -txfC0

6. Получение ответа (количество CAN сообщений) на отправленное CAN сообщение, ключ -rcnt
>canscan.exe -msg0000012355558c442a008c44 -rdt100 -wrt100 -rcnt1
В данном примере будет отправлено CAN сообщение [00000123] 55 55 8c 44 2a 00 8c 44 и выведен ответ ЭБУ на него

Все ключи можно компоновать в произвольном виде и порядке
Возможно создавать скрипт в виде bat файла для гибкой работы с CAN шиной в пакетном режиме
Пример файла test.bat :
canscan.exe -msg0000012355558c442a008c44 -rdt100 -wrt100 -rcnt1
canscan.exe -msg000012341122334455667788 -wrt200 -wcnt50 -per200

Если программа CanScan завершилась не по нажатию клавиш Ctrl-C, то возвращается 0 exit code, в противном случае он будет не нулевым
